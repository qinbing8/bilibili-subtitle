# 快速开始

这份文档给第一次接手这个项目的人一条最短路径：先跑通本地，再决定是否部署到 Vercel。

如果你只想知道完整部署细节，看 [DEPLOYMENT.md](./DEPLOYMENT.md)；如果你遇到本地联调问题，看 [MAINTENANCE.md](./MAINTENANCE.md)。

## 你会得到什么

跑通后，页面会完成下面这条链路：

`Bilibili URL -> /api/audio-proxy -> Tongyi Tingwu -> .docx / .txt`

注意：

- 前端不会再拿到完整带 token 的 `proxyUrl`
- 听悟只能访问公网 `http(s)` 地址，不能直接访问你的 `localhost:9091`

## 前置要求

- Node.js `24.x`
- npm `11+`
- 阿里云通义听悟相关密钥
- 通义千问 `DASHSCOPE_API_KEY`
- 一个可访问的公网代理地址
  - 本地联调推荐 `cloudflared`
  - Vercel 部署可先直接用站点域名

## 路径 A：本地 10 分钟跑通

### 1. 安装依赖

```powershell
Set-Location D:\workspeace\bilibili-subtitle
npm install
```

### 2. 准备环境变量

复制 `.env.example` 为 `.env.local`，至少填这些：

```text
ALI_ACCESS_KEY_ID=
ALI_ACCESS_KEY_SECRET=
ALI_APP_KEY=
DASHSCOPE_API_KEY=
APP_ACCESS_PASSWORD=
PUBLIC_PROXY_BASE_URL=
AUDIO_PROXY_TOKEN_SECRET=
```

建议再补：

```text
BILIBILI_SESSDATA=
META_TOKEN_SECRET=
```

### 3. 启动公网 tunnel

本地后端不能被听悟直接访问，所以先在独立终端运行：

```powershell
cloudflared tunnel --url http://localhost:9091
```

把输出的 `https://*.trycloudflare.com` 填到 `.env.local` 的 `PUBLIC_PROXY_BASE_URL`。

### 4. 启动后端

仍然在独立终端运行：

```powershell
Set-Location D:\workspeace\bilibili-subtitle
npm run dev:backend
```

期望看到：

```text
Server running at http://localhost:9091
```

### 5. 启动前端

再开一个终端：

```powershell
Set-Location D:\workspeace\bilibili-subtitle
npm run dev:frontend
```

打开 `http://localhost:5173`，输入 `APP_ACCESS_PASSWORD`。

### 6. 做最小验证

```powershell
npm run check
npm test
```

然后验证两个健康接口：

```powershell
Invoke-RestMethod -Uri 'http://127.0.0.1:9091/api/health'
Invoke-RestMethod -Uri 'http://127.0.0.1:9091/api/health-with-config'
```

你应该看到：

- `/api/health` 返回 `{"status":"ok"}`
- `/api/health-with-config` 返回 `proxyBaseUrlConfigured: true`

### 7. 提交一个真实任务

进入页面后：

1. 输入密码
2. 粘贴一个 B 站视频链接
3. 点击“提交转写任务”
4. 等待状态从 `ONGOING` 进入 `COMPLETED`
5. 下载 `.docx` 或 `.txt`

## 路径 B：直接部署到 Vercel

### 1. 推送代码到 GitHub

```powershell
git push origin main
```

如果仓库还没连 GitHub，先在 GitHub 建仓库，再补 `remote` 后推送。

### 2. 在 Vercel 导入项目

1. 打开 https://vercel.com
2. 选择 `New Project`
3. 导入这个仓库
4. 保持默认 Vite 构建设置
5. 在 Environment Variables 中填入 `.env.example` 对应变量

当前建议至少填：

- `ALI_ACCESS_KEY_ID`
- `ALI_ACCESS_KEY_SECRET`
- `ALI_APP_KEY`
- `DASHSCOPE_API_KEY`
- `APP_ACCESS_PASSWORD`
- `AUDIO_PROXY_TOKEN_SECRET`
- `PUBLIC_PROXY_BASE_URL=https://<your-project>.vercel.app`

### 3. 部署后做最小验收

1. 首页先出现密码登录遮罩
2. 输入密码后可进入主界面
3. 提交 B 站链接后，任务成功创建
4. 任务完成后可下载 `.docx` / `.txt`

## 已知限制

### 1. Vercel 有 300 秒函数时长限制

当前 `vercel.json` 对 `api/index.ts` 配置的是：

```json
{
  "functions": {
    "api/index.ts": {
      "maxDuration": 300
    }
  }
}
```

所以特别长的音频流可能在 Vercel 侧被超时截断。

### 2. 音频下载成功不代表听悟一定能读

常见失败原因不是 B 站接口坏了，而是公网代理没配对。优先检查：

- `PUBLIC_PROXY_BASE_URL` 是否是公网地址
- tunnel 是否仍在运行
- `AUDIO_PROXY_TOKEN_SECRET` 是否已配置
- `BILIBILI_SESSDATA` 是否已配置

### 3. 当前前端只展示 host

这是安全设计，不是 bug。页面只显示：

- `audioHost`
- `proxyHost`

不会再把完整代理 URL 暴露给浏览器。
